# Before / after examples

Worked rewrites covering each category of tell. The principle in every case: change as little as possible while removing the giveaway. Don't impose a new voice.

## Full-spectrum marketing slop

**Before:**

> In today's rapidly evolving technological landscape, it's not just about building software — it's about crafting experiences that truly resonate with users. Our team's commitment to excellence, combined with our robust engineering practices, serves as a testament to our enduring dedication to delivering groundbreaking solutions. It's worth noting that we leverage cutting-edge technologies to navigate the complexities of modern development. In conclusion, our holistic approach empowers teams to thrive in an ever-changing world.

**After:**

> We build software people want to use. Our team is good at engineering and we care about the work. We use modern tools because they help us ship.

*Heavy rewrite — original meaning preserved but most sentences restructured.*

Most of the input was empty. The rewrite says what the original would have said if it had been written by someone with nothing to prove.

## Overused vocabulary cluster

**Before:**

> The platform's robust architecture empowers teams to seamlessly leverage cutting-edge tools, fostering a vibrant ecosystem of innovation.

(Five flagged words in one sentence: robust, empowers, seamlessly, leverage, fostering, vibrant.)

**After:**

> The platform lets teams use modern tools without friction, which keeps the ecosystem active.

## Em-dash overuse

**Before:**

> The new release is faster — much faster — and includes a redesigned editor — one built from scratch for performance — plus a host of smaller improvements.

(Four em-dashes in one sentence.)

**After:**

> The new release is much faster. It includes a redesigned editor (built from scratch for performance) and a host of smaller improvements.

## Hedging filler

**Before:**

> It's worth noting that the API now supports streaming responses. It goes without saying that this is a significant improvement.

**After:**

> The API now supports streaming responses. This is a significant improvement.

## Recap outro

**Before:**

> The migration touched twenty-three services and ran for six weeks without an outage. In conclusion, careful planning paid off.

**After:**

> The migration touched twenty-three services and ran for six weeks without an outage. Careful planning paid off.

## Bullet-and-bold soup → prose

**Before:**

> The framework offers several benefits:
> - **Performance**: Fast rendering across all devices.
> - **Flexibility**: Adapts to a wide range of use cases.
> - **Developer experience**: Clean API surface.

**After:**

> The framework is fast on all devices, adapts to a wide range of use cases, and has a clean API.

## Elegant variation

**Before:**

> The author opens with a striking metaphor. The writer then pivots to a personal anecdote. The novelist closes with a return to the metaphor.

(All three sentences are about the same person.)

**After:**

> The author opens with a striking metaphor, pivots to a personal anecdote, then closes with a return to the metaphor.

## Clean text (no changes needed)

*No changes — text reads as written by a human.*
